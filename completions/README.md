# completions

Generated shell completion scripts for `pg_hardstorage`. Shipped by every OS
package.

## What lives here

One subdirectory per supported shell. Files are **auto-generated by `make
docs-regen`; do not edit by hand** — the generator (`../cmd/docsgen/`) emits
them from the Cobra command tree in `../internal/cli/`, so completions stay in
lock-step with the CLI surface. Edits are clobbered on the next regen.

## Key files / subdirs

- `bash/pg_hardstorage` — bash completion (sourced by `bash-completion` or
  installed under `/usr/share/bash-completion/completions/`)
- `zsh/_pg_hardstorage` — zsh completion (installed under
  `/usr/share/zsh/site-functions/`)
- `fish/pg_hardstorage.fish` — fish completion (installed under
  `/usr/share/fish/vendor_completions.d/`)

## Read next

- `../cmd/docsgen/` — generator entry point; `make docs-completions` is the
  targeted Make recipe
- `../internal/cli/` — Cobra commands that are the source of truth for
  available flags and subcommands
- `../debian/pg-hardstorage.bash-completion` — Debian install manifest hook

## Don't put X here

- Hand-edited completion scripts — the next `make docs-regen` will overwrite
  them.
- Completions for other shells (PowerShell, nushell, etc.) — extend
  `../cmd/docsgen/` first, then regenerate.

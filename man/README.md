# man

Generated troff man pages, one per CLI subcommand. Shipped by every OS package
and installed under `/usr/share/man/man1/`.

## What lives here

Section-1 man pages for the `pg_hardstorage` CLI tree — one file per command
(e.g. `pg_hardstorage-backup-compare.1`, `pg_hardstorage-audit-search.1`). Files
are **auto-generated by `make docs-regen`; do not edit by hand** — edits are
clobbered on the next regen. The generator lives in `../cmd/docsgen/` and pulls
metadata from the Cobra command tree in `../internal/cli/`.

## Key files / subdirs

- `man1/` — section-1 man pages, one per leaf subcommand (e.g.
  `pg_hardstorage-backup-compare.1`, `pg_hardstorage-agent.1`,
  `pg_hardstorage-anomaly-check.1`)

## Read next

- `../cmd/docsgen/` — the generator that emits this tree
- `../internal/cli/` — Cobra command definitions that are the source of truth
- `../debian/pg-hardstorage.manpages` — Debian install manifest for these
  files
- `../docs/reference/cli/` — Markdown counterpart of the same content

## Don't put X here

- Hand-edited man pages — they will be overwritten by the next `make
  docs-regen` run.
- Sections other than 1 — pg_hardstorage exposes only user commands; no
  daemons, no syscalls.
- Localised man pages — i18n is `../internal/i18n/` material; man pages stay
  English.
